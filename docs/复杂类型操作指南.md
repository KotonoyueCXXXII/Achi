# 复杂类型与指针操作指南

## 指针是什么

HPL（Happy2018new's Python-AST Language）只原生支持 `int`/`float`/`str`/`bool` 四种类型。`map`、`slice`、`tuple`、`set` 等复杂类型通过**指针**（整数句柄）间接操作。

底层实现：全局 `dict<int, object>`，指针的整数值就是句柄，作为 key 查找对应的对象。`maps.new`、`slices.new` 等函数返回的都是这个整数。

**为什么用指针而不是原生语法**（如 `[1,2,3]`、`{'k':'v'}`）？原生支持新类型需要修改语言解析器——每加一种类型就要动语法。而指针方案只需增加内置函数，无需改语言本身。这是开发者在语言已接近完成时做的务实取舍。

## 指针生命周期

指针通过 `maps.new`、`slices.new`、`tuple.new`、`set.new` 等创建，返回整数指针，底层存入全局 `dict<int, object>`。

```
maps.new / slices.new / tuple.new / set.new
      │
      ▼
  指针创建（整数句柄，存入全局 dict）
      │
      ├──→ 仅在函数内使用 ──→ 函数退出时自动释放
      │
      └──→ 需要传出函数
              │
              ▼
         pin(ptr)   ← 函数内
              │
              ▼
         return ptr ← 函数内
              │
              ▼
         调用方使用完毕
              │
              ▼
         finalise(ptr) ← 调用方
```

两条路径：

- **仅函数内使用**：函数退出时自动释放，无需任何操作。
- **需要传出函数**：函数内 `object.pin(ptr)` → `return ptr`。**调用方**使用完毕后由调用方执行 `object.finalise(ptr)`，不在被调函数内 finalise。

| 操作 | 作用 | 适用场景 |
|---|---|---|
| `object.pin(ptr)` | 阻止自动释放 | 把指针传出函数时调用 |
| `object.finalise(ptr)` | 撤销 pin | 用完 pin 过的指针后释放 |
| `object.release(ptr)` | 立即删除指针 | 临时指针（如 `world.SetCommand`）用完即弃 |

`finalise` 撤销的是 `pin` 的保护，`release` 是直接删除指针。两者不对等，不能互换。

**pin 的有效范围：** 现阶段 pin 执行后，该指针在同一游戏 tick 内整个调用栈上都无法被回收（除非手动 `release`）。**这不是标准特性，未来可能变更，不可依赖。**

**释放只失效句柄：** `release` 只是删除全局 dict 中的 key（即句柄），目标对象本身可能被其他地方持有（如 ExtraData），不会因此销毁。GC 是一个简单的定时轮询器，延后批量回收未引用的对象。

**解引用限制：** 大多数 `world.*`/`entity.*` API 返回的是指针（如 `entity.EntityHasTag`），需先用 `object.deref(ptr)` 转为原始值才能用于 `if` 等条件判断。只有指向 int/bool/float/str 的指针可解引用——指向 None 的指针（如传入无效 entityId）解引用会失败，需先 `object.is_none(ptr)` 检查。



## ptr_set：存入复杂类型

所有复杂类型模块的写操作都分 `set` 和 `ptr_set` 两套。**存入复杂类型必须用 `ptr_set` 或等价函数**——`set` 只存原始值（int/str/bool），指针传进去会被当成普通整数存储，原指针释放后变成悬垂引用（实际类型为 int）。

```hpl
# 错误：maps.set 存指针 → 函数返回后悬垂引用（实际为int）
_ = {func, maps.set(outerMap, 'key', innerMap)}

# 正确：maps.ptr_set 存指针
_ = {func, maps.ptr_set(outerMap, {func, object.ref('key')}, innerMap)}
```

标量键需包装为指针：`{func, object.ref('stringKey')}` 或 `{func, object.ref(123)}`。

## ptr_get：读取复杂类型

读回用 `ptr_get`，返回的是**调用时刻的快照**，非实时引用。后续修改原 map 不会反映到已有返回值中。

**嵌套写入必须"先填子 map、再存父 map"：**

```hpl
# 1. 取出子 map（快照）
child = {func, maps.ptr_get(parent, {func, object.ref('childKey')})}

# 2. 修改子 map
_ = {func, maps.ptr_set(child, {func, object.ref(0)}, newValue)}

# 3. 存回父 map —— 必须！ptr_get 是快照，不存回则修改丢失
_ = {func, maps.ptr_set(parent, {func, object.ref('childKey')}, child)}
```

> **例外：** `entity.GetExtraData` 返回的是指向 ExtraData 内部对象的实时引用，非快照。直接 `ptr_set` 修改即可持久化，无需再调用 `SetExtraData`。

```hpl
data_ptr = {func, entity.GetExtraData(worldId, key)}
_ = {func, maps.ptr_set(data_ptr, {func, object.ref('key')}, valuePtr)}
```

`ptr_get` 适用于所有复杂类型：map、slice、tuple、set。`entity.GetExtraData` 同理。


## 常用代码模式

### maps.get 前必须 maps.exist 守卫

```hpl
# 正确
if {func, maps.exist(data, 'key')}:
    value = {func, maps.get(data, 'key')
fi

# 错误：不判断直接 get → key 不存在时崩溃
value = {func, maps.get(data, 'maybeMissing')}
```

### 用 ref_type 判断复杂类型 key 是否存在

```hpl
child = {func, maps.ptr_get(parent, {func, object.ref('key')})}
if {func, object.ref_type(child)} == 6:    # 6 = map
    # key 存在
fi
```

不用 `maps.ptr_exist` 预检——`ptr_get` 取出后直接判类型更简洁。

### 用 maps.del / ptr_del 删除，禁止伪装删除

```hpl
# 正确
_ = {func, maps.del(mapPtr, 'key')}
_ = {func, maps.ptr_del(mapPtr, {func, object.ref('key')})}

# 错误：用 set 哨兵值伪装删除
_ = {func, maps.set(mapPtr, 'key', -2147483648)}
```

### 遍历 map 的 key-value

```hpl
items = {func, maps.items(myMap)}
length = {func, slices.length(items)}
for i, length:
    item = {func, slices.ptr_get(items, i)}
    key = {func, tuple.get(item, 0)}
    value = {func, tuple.get(item, 1)}
rof
```

### 逐层 ptr_get 读取嵌套结构

```hpl
child = {func, maps.ptr_get(parent, {func, object.ref('childKey')})}
if {func, object.ref_type(child)} == 6:
    grandchild = {func, maps.ptr_get(child, {func, object.ref('grandchildKey')})}
    if {func, object.ref_type(grandchild)} == 7:
        # 使用 grandchild...
    fi
fi
```

### 哨兵检查后 finalise

```hpl
ptr = {func, function.call('data/ptrMapGet', 'domain', key)}
if ptr == -2147483648:
    return    # 绝不 finalise 哨兵
fi
# 使用 ptr...
_ = {func, object.finalise(ptr)}
```

## 类型码速查

| `object.ref_type(ptr)` | `object.raw_type(val)` | 类型 |
|---|---|---|
| `0` | `0` | int |
| `1` | `1` | bool |
| `2` | `2` | float |
| `3` | `3` | str |
| `4` | — | None |
| `5` | — | slice |
| `6` | — | map |
| `7` | — | tuple |
| `8` | — | set |
| `0xFFFF` | — | 无效指针 |

`ref_type` 查指针指向的类型，`raw_type` 查原始值的类型（仅支持 0–3）。
