文件重构计划：

这是什么

这是一套以本地联机：创造模式为核心场景而设计的玩家管理工具。
本系统的核心功能是一套地皮管理机制。

地皮是什么

在一个世界中，地皮为每一位玩家定义了一个独立、固定边界、可自由建设与管理的专属地块。

我们为什么需要地皮

地皮这个机制能够隔离玩家与玩家之间的建筑干扰，规避不必要的冲突。典型的场景就是创造自建服的防熊需求。

与其他地皮插件的区别是什么

市面上其他的地皮插件主要有以下两种形式：

1：基于生存模式的领地概念而开发的插件，他们的实现依靠玩家自己的能动性。
  优点：地皮的位置尺寸全靠玩家自决定，自由度很高。
  缺点：我们难以判断玩家所选择的区域是否合法，且地皮往往分散，难以执行建筑托管。

2：完全依靠系统算法自动生成的地皮。
  优点：自动化，不需要过多干预，地皮整齐排列，为每个玩家自动分配。
  缺点：过于整齐，自由度相对较低，我们不能替玩家确定他应该在哪里建筑。

我们的理念

我们想要设计更自由也更规范的地皮系统，既可以保证玩家的选择自由度，又可以方便管理员进行建筑区域规划。
于是我构思了一套动态分配的地皮系统。

核心理念：玩家对地皮只有使用权，没有所有权。地皮是管理员预设的固定资源池。

玩家 A 首次进入
      │
      ▼

      │
      ▼

      │
      ▼
玩家 A 退出
      │
      ▼
建筑存档至远程备份区 ─────────────────┐
地皮释放，归还到空闲池                 │
      │                                 │
      ▼                                 │
玩家 B 申请同尺寸地皮                   │
      │                                 │
      ▼                                 │
复用玩家 A 释放的地皮                   │
（清空地块 → B 从头建造）               │
      │                                 │
      ▼                                 │
玩家 A 再次上线                         │
      │                                 │
      ▼                                 │
只能匹配上次存档的尺寸                   │
      │                                 │
      ▼                                 │
分配合适的空闲地皮 ← 可以是完全不同的地块  │
      │                                 │
      ▼                                 │
从远程备份区加载建筑 ←──────────────────┘
      │
      ▼
恢复完成，玩家 A 继续建造

这样做的好处

1. 管理员可随时调整地皮的位置，想让玩家建哪里就建哪里。
2. 取消了玩家与地皮的一对一绑定关系：用最少的地皮，供最多的玩家使用。防止刚分配到地皮就退出的玩家占用地皮资源。
3. 建筑持久化：玩家离线后建筑被完整保存，再次上线自动恢复。玩家无需担心建筑丢失。
4. 池化资源复用：一块地皮可在不同时段服务于多名玩家，大幅降低服务器世界中的地皮总占用面积。
5. 管理员驱动布局 + 玩家自由选择：管理员划定"可以在哪里建"，玩家决定"我要多大的地"。两全其美。


系统主要功能

地皮管理
  - 管理员创建/删除地皮（三种尺寸：20×20 / 30×30 / 40×40）
  - 玩家按需领取空闲地皮，系统自动分配
  - 玩家离线自动备份建筑，上线自动恢复

建筑保护
  - 四级权限模型：访客(0) / 成员(1) / 会员(2) / 管理员(3)
  - 成员仅能在自己的地皮范围内建造，会员全局放行，管理员拥有全部权限
  - 拦截方块破坏、方块放置事件，按权限实时判定

备份与恢复
  - 玩家离线时触发异步备份队列，将建筑克隆至远程存储区
  - 基于玩家 UID 的螺旋分配算法，均匀分布备份数据
  - 玩家上线后自动匹配同尺寸地皮并恢复建筑

传送系统
  - 7 个行政定点（主城、广场、专区等）
  - 每名玩家 9 个自定义锚点，可随时写入/读取/删除坐标
  - 一键传送至自己的地皮

玩家交互
  - 在线玩家列表，支持传送/私信/管理操作
  - 自定义聊天格式，按权限等级显示彩色前缀
  - 管理员可踢出玩家、解绑地皮、调整权限

系统特性
  - 命令拦截：仅管理员可执行 / 命令
  - 26 个 UI 表单，完整菜单导航
  - 每 30 秒自动持久化，防止数据丢失


术语约定

  地皮: 管理员预设的固定位置、固定尺寸的可建筑区域
  分配: 将空闲地皮临时绑定给玩家使用（非所有权转移）
  备份/恢复: 玩家离线时建筑克隆至远程存储，上线时克隆回来
  尺寸索引: 0=20×20, 1=30×30, 2=40×40
