# 指令概览
```mcfunction
compilecache compile <code: string>
compilecache query
compilecache set [size: int]
```





# 概述
为了提升在执行代码时的性能，我们引入了缓存系统。<br/>
具体来说，在引入缓存系统前，您的源代码需要在执行前被解析和翻译为程序可以读懂的“语言”，然而这一过程将消耗大量的时间。

然而我们发现，一旦解析和翻译工作完成，并且以后都始终使用这样的结果产物来执行代码，可以让代码执行效率提高二十倍以上。<br/>
这是很容易理解的，因为每次执行代码时可以跳过源代码解析和翻译的过程，而是转而使用之前的解析和翻译结果来执行代码。

**代码缓存系统**则存储了这些已经解析和翻译好的产物，于是我们可以提高执行代码时的性能。<br/>
然而，**代码缓存系统**不可能是无限大的，但您可以通过 `compilecache` 指令来动态调整其大小。





# 编译代码
## 语法
解析和翻译 `<code: string>` 所指示的源代码，并将处理结果储存到代码缓存系统。<br/>
从学术上讲，这个过程先将源代码处理为抽象语法树，然后编译为字节码，最后放入缓存中。

```mcfunction
compilecache compile <code: string>
```



## 示例
编译下方的代码，并将编译产物放入缓存系统。
```python
total = -2018
for i, 32:
  total = total + 10*i
rof
return total
```

对应的指令如下。
```mcfunction
compilecache compile "total = -2018
for i, 32:
  total = total + 10*i
rof
return total"
```





# 查询缓存系统的状态
## 语法
查询代码缓存系统已经使用的部分，以及它的最大容量。
```mcfunction
compilecache query
```



## 效果（示例）
```
预编译缓存区已存在 4 个缓存，最多存在 2048 个
```





# 设置最大容量
## 语法
设置代码缓存系统的最大容量为 `[size: int]`，或重置为默认值。
```mcfunction
compilecache set [size: int]
```



## 备注
- 代码缓存系统的默认最大容量为 2048
- 如果没有特别的需要，请勿手动调整最大容量
- 最大容量过小可能会导致代码执行效率下降
- 最大容量过大可能会导致服务器内存占用过高



## 示例
```mcfunction
# 将代码缓存系统的最大容量设置为 114514
compilecache set 114514

# 将代码缓存系统的最大容量设置为 5201314
compilecache set 5201314

# 将代码缓存系统的最大容量设置为默认值（2048）
compilecache set
```
