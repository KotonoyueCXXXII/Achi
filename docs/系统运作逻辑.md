# 系统运作逻辑

## 启动

`.mcfunction` 导入时输出安装信息（`tellraw @a`），随即注册 `init/main` 并通过 `customfunction call` 触发，完成所有 ExtraData 域的初始化。

首位玩家加入时，`player_join` 检测到服务器此前为空，遍历 `temp.land.using` 将异常关闭残留的地皮逐个入队备份，随后清理并重建临时域。

## 主循环

`main` 每游戏 tick（~20Hz）执行，通过 `data.main.schedule` 计数器调度不同频率的任务：

```
每 tick：
  ├── titleraw @a[tag=isSettingLand]     ← 显示地皮设置提示
  ├── execute ... ui/openMenu              ← 检测低头玩家，弹出菜单
  └── schedule++

每 5 tick（~4Hz）：
  ├── land/processBackup         ← 驱动备份队列
  └── land/processInit           ← 驱动地皮初始化队列

每 600 tick（~30s）：
  ├── entity.SaveExtraData       ← 持久化存档
  └── customform style @a        ← 刷新表单样式
```

## 地皮生命周期

| 阶段 | 触发 | 操作 |
|---|---|---|
| 创建 | 管理员 `ACHI_HOME_LAND_SET` | 选尺寸 → 点击地面选西北角 → `item_use` 捕获坐标 → 确认写入 `data.land.archive` |
| 分配 | 玩家 `ACHI_HOME_LAND_GET` | `land/getFreeIndex` 扫描空位 → 绑定 `temp.land.using` → 传送 → `queue.land.init` 入队 |
| 初始化 | `main` 每 5 tick | `land/processInit` 递减 retryCount → 等区块 → `land/initLand` 清空+铺草皮+标记 |
| 使用 | `item_use` / `block_destroy` | 按 privilegeLevel 拦截：管理员放行，成员仅自己地皮，访客全拦 |
| 离开 | `player_leave` 事件 | `queue.land.backup` 入队（mode=0）→ 删 `temp.player.index` → `temp.land.using` 保留作锁 |
| 备份 | `main` 每 5 tick | `land/processBackup` state 0 计算坐标 → state 1 clone → 成功写 `saveSizeIndex` 并解锁 |
| 恢复 | 玩家再次上线 | 检测 `saveSizeIndex` → `ACHI_INFO_RESTORE` 确认 → 分配同尺寸地皮 → 入队（mode=1）→ clone 回地皮 |
| 删除 | 管理员 `ACHI_INFO_LAND_DEL` | 站地皮上 → 坐标匹配 → 确认后解绑 + swap-and-pop |

```
管理员创建地皮  .........  ACHI_HOME_LAND_SET → item_use → ACHI_INFO_LAND_SET
      │
      ▼
玩家加入  .........  player_join
      │
      ├──→ 有存档？──→ land/getFreeIndex → queue.land.backup(mode=1) → land/processBackup → land/cloneLand
      │
      ▼
分配空闲地皮  .........  ACHI_HOME_LAND_GET → land/getFreeIndex
      │
      ▼
初始化入队  .........  queue/push → queue.land.init
      │
      ▼
初始化执行  .........  land/processInit → land/checkChunks → land/initLand → queue/pop
      │
      ▼
玩家建造  .........  item_use / block_destroy（权限拦截）
      │
      ├──→ 地皮解绑（管理员 / 备份超时） .........  ACTION.hpl → 删 saveSizeIndex → 删 temp.land.using
      │
      ▼
玩家离开  .........  player_leave
      │
      ▼
备份入队  .........  queue/push → queue.land.backup(mode=0)
      │
      ▼
备份执行  .........  land/processBackup → land/squareSpiralFill → land/cloneLand → queue/pop
      │
      ▼
备份完成  .........  写 saveSizeIndex → 删 temp.land.using → 地皮可重新分配

管理员删除地皮  .........  HOME/LAND/_.hpl → ACHI_INFO_LAND_DEL → 解绑 → swap-and-pop → data.land.archive 移除
```

## 玩家生命周期

| 阶段 | 触发 | 函数链 |
|---|---|---|
| 加入 | `player_join` 事件 | 分配 `temp.player.index` 槽位 → 空服则清理残留。默认访客（privilegeLevel=0），无法建造 |
| 同意协议 | `ACHI_INFO_AGREEMENT`，点击"同意" | 已有账户 → 设 `privilegeLevel=1`；新玩家 → `data.main.nextUID++` → 创建 `data.player.archive`（privilegeLevel=1） |
| 晋升会员 | 管理员 `ACHI_HOME_PLAYER_ADMIN` | `privilegeLevel` 设为 2 → 全局建造权限 |
| 晋升管理员 | `ACHI_HOME_SYSTEM_PERSONAL` 输入密码 | `privilegeLevel` 设为 3 → 最高权限 |
| 离开 | `player_leave` 事件 | 入队备份 → 删 `temp.player.index` |

```
加入（访客）  .........  player_join → privilegeLevel=0
      │
      ▼
同意协议（成员）  .........  ACHI_INFO_AGREEMENT → privilegeLevel=1
      │
      ├──→ 晋升会员  .........  ACHI_HOME_PLAYER_ADMIN → privilegeLevel=2
      │
      ├──→ 晋升管理员  .........  ACHI_HOME_SYSTEM_PERSONAL → privilegeLevel=3
      │
      ▼
离开  .........  player_leave → 入队备份 → 删 temp.player.index
```

## 队列机制

耗时操作不阻塞主循环，通过队列异步推进。队列任务完整字段见 [数据结构参考.md](数据结构参考.md)。

### 队列操作

- `queue/push`：线性扫描 0–63 槽位，写入首个空位
- `queue/peek`：读取槽位 0 并 pin，**需 finalise**
- `queue/pop`：删除槽位 0，前移后续元素

### 备份队列（`queue.land.backup`）

由 `land/processBackup.hpl` 每 5 tick 处理槽位 0，分两个状态：

**State 0（准备）：**
- 读取 landIndex → 获取地皮数据
- `land/squareSpiralFill(playerUID)` 计算螺旋备份偏移
- 基坐标 `(1048576, 1048576)`，每格 40 区块
- 在源/目标创建 tickingarea
- 写入 `src_*` / `dst_*` / `sizeIndex`
- 进入 state 1

**State 1（执行）：**
- `retry++`，最多 12 次（约 3 秒）
- 等待区块加载 → `land/cloneLand` 纵向分段 clone（Y=-64 至 128，每段最多 81 层）
- 成功：清理 tickingarea → mode=0 时写入 `saveSizeIndex` → `queue/pop`
- 超时：清理 → `queue/pop`

### 初始化队列（`queue.land.init`）

由 `land/processInit.hpl` 每 5 tick 处理槽位 0：

1. 递减 `retryCount`（起始 15）
2. 归零则超时弹出，放弃
3. 否则 `land/checkChunks` 检查区块加载
4. 就绪后 `land/initLand`：逐段 fill air 清空 → fill grass 铺草皮 → setblock 绿色玻璃标记 → 传送玩家 → `queue/pop`

## 地皮创建（管理员）

1. 管理员进入 `ACHI_HOME_LAND_SET` 选择尺寸
2. `temp.land.setting[playerIndex]` 写入 `{ sizeIndex }`
3. 游戏提示点击地面选择西北角
4. `item_use` 事件捕获坐标，补全 `anchor_x/y/z`
5. `ACHI_INFO_LAND_SET` 确认，写入 `data.land.archive`
6. 在四角放置绿色玻璃标记

## 地皮删除（管理员）

1. 管理员站到要删除的地皮上
2. `HOME/LAND/_.hpl` 遍历 `data.land.archive`，通过坐标匹配找到地皮
3. `temp.land.deleting[playerIndex] = landIndex`
4. `ACHI_INFO_LAND_DEL` 确认后：检查备份队列 → 解除 `temp.land.using` 绑定 → swap-and-pop 删除记录

## 螺旋备份算法

`land/squareSpiralFill(uid)` 根据玩家 UID 计算螺旋坐标偏移，使备份在 `(1048576, 1048576)` 基坐标周围均匀分布，避免热点。
