# Achi — 地皮管理系统

## 📦 这是什么

这是一套以**本地联机：创造模式**为核心场景而设计的玩家管理工具。本系统的核心功能是一套地皮管理机制。

## 🏠 地皮是什么

在一个世界中，地皮为每一位玩家定义了一个**独立、固定边界、可自由建设与管理**的专属地块。

## ❓ 我们为什么需要地皮

地皮这个机制能够隔离玩家与玩家之间的建筑干扰，规避不必要的冲突。典型的场景就是创造自建服的防熊需求。

## ⚖️ 与其他地皮插件的区别是什么

市面上其他的地皮插件主要有以下两种形式：

**类型一：基于生存模式的领地插件**
- 优点：地皮的位置尺寸全靠玩家自决定，自由度很高
- 缺点：难以判断玩家所选择的区域是否合法，且地皮往往分散，难以执行建筑托管

**类型二：完全依靠系统算法自动生成的地皮**
- 优点：自动化，不需要过多干预，地皮整齐排列，为每个玩家自动分配
- 缺点：过于整齐，自由度相对较低，不能替玩家确定他应该在哪里建筑

## 💡 我们的理念

我们想要设计更自由也更规范的地皮系统，既可以保证玩家的选择自由度，又可以方便管理员进行建筑区域规划。于是构思了一套**动态分配的地皮系统**。

**核心理念：玩家对地皮只有使用权，没有所有权。地皮是管理员预设的固定资源池。**

```
玩家上线
    │
    ├──→ 首次：阅读协议 → 注册账户
    │
    ▼
匹配空闲地皮
    │
    ▼
清空区域、铺草皮、放置标记
    │
    ├──→ 有存档：从备份区恢复建筑
    │
    ▼
在地皮范围内自由建造
    │
    ▼
玩家下线
    │
    ▼
建筑存档，地皮归还资源池
```

## ✅ 这样做的好处

1. **管理员可随时调整地皮的位置**，想让玩家建哪里就建哪里
2. **取消一对一绑定**：用最少的地皮，供最多的玩家使用，防止刚分配到地皮就退出的玩家占用地皮资源
3. **建筑持久化**：玩家离线后建筑被完整保存，再次上线自动恢复，无需担心建筑丢失
4. **池化资源复用**：一块地皮可在不同时段服务于多名玩家，大幅降低世界中的地皮总占用面积
5. **管理员驱动布局 + 玩家自由选择**：管理员划定"可以在哪里建"，玩家决定"我要多大的地"，两全其美

## 🔧 系统主要功能

**🏗️ 地皮管理**
- 管理员创建/删除地皮（三种尺寸：20×20 / 30×30 / 40×40）
- 玩家按需领取空闲地皮，系统自动分配
- 玩家离线自动备份建筑，上线自动恢复

**🛡️ 建筑保护**
- 四级权限模型：访客(0) / 成员(1) / 会员(2) / 管理员(3)
- 成员仅能在自己的地皮范围内建造，会员全局放行，管理员拥有全部权限
- 拦截方块破坏、方块放置事件，按权限实时判定

**💾 备份与恢复**
- 玩家离线时触发异步备份队列，将建筑克隆至远程存储区
- 基于玩家 UID 的螺旋分配算法，均匀分布备份数据
- 玩家上线后自动匹配同尺寸地皮并恢复建筑

**🌐 传送系统**
- 7 个行政定点（主城、广场、专区等）
- 每名玩家 9 个自定义锚点，可随时写入/读取/删除坐标
- 一键传送至自己的地皮

**👥 玩家交互**
- 在线玩家列表，支持传送/私信/管理操作
- 自定义聊天格式，按权限等级显示彩色前缀
- 管理员可踢出玩家、解绑地皮、调整权限

**⚙️ 系统特性**
- 命令拦截：仅管理员可执行 `/` 命令
- 26 个 UI 表单，完整菜单导航
- 每 30 秒自动持久化，防止数据丢失

## 🛠️ 项目实现

- **语言**：HPL（Happy2018new's Python-AST Language），网易 Minecraft 专有脚本语言
- **构建**：Python 构建脚本，将全部 `.hpl` 源文件压缩拼接为单一 `Achi.mcfunction`（~100KB）
- **运行时**：事件驱动 + 主循环（每 tick 执行），耗时操作异步队列化
- **存储**：14 个 ExtraData 域，三段式命名 `类型.模块.属性`：
  - `data.*` — 持久层（玩家账户、地皮注册表、全局状态）
  - `temp.*` — 临时层（在线玩家索引、活跃地皮绑定、菜单状态）
  - `queue.*` — 队列层（地皮初始化、备份状态机）
- **权限**：0=访客、1=成员、2=会员、3=管理员
- **入口**：6 个系统事件 + 28 个自定义函数 + 26 个 UI 表单

## 🚀 快速开始

```bash
# 构建（输出单一 Achi.mcfunction，上传至 MC 服务器）
uv run python build.py
```

构建输出：`Achi.mcfunction`（~100KB），导入服务器即可运行。卸载脚本：`Achi_uninstall.mcfunction`。

## 📋 术语约定

| 术语 | 说明 |
|---|---|
| 地皮 | 管理员预设的固定位置、固定尺寸的可建筑区域 |
| 分配 | 将空闲地皮临时绑定给玩家使用（非所有权转移） |
| 备份/恢复 | 玩家离线时建筑克隆至远程存储，上线时克隆回来 |
| 尺寸索引 | 0=20×20, 1=30×30, 2=40×40 |
| 指针 | HPL 对象引用句柄（整数），通过 `object.ref_type` 等函数操作 |
| pin/finalise | 钉住指针防止自动回收 / 释放已钉住的指针 |
| ExtraData | HPL 持久化/临时数据存储，三段式命名 `类型.模块.属性` |
| 目标选择器 | `@a`（所有玩家）、`@p`（最近玩家）、`@s`（自身） |
